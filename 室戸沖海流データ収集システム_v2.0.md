# 室戸沖 海流データ収集・可視化システム

**バージョン: 2.0**
**作成日: 2026年4月12日**
**ステータス: 稼働中（データ収集自動化＋HTMLダッシュボード完成）**

---

## 概要

室戸岬沖（室戸沖）の海流データを2022年1月1日以降の日次データとして継続的に収集・蓄積するPythonシステムと、収集データをブラウザ上で可視化するHTMLダッシュボードのセット。

**v1.0 → v2.0 の主な変更点:**
- HTMLダッシュボード（`muroto_dashboard.html`）の新規開発
- データ変換スクリプト（`update_dashboard_data.py`）の新規開発
- ブラウザ上でのグラフ表示・テーブル閲覧・欠損チェック機能

---

## v2.0 新規開発内容

### 1. HTMLダッシュボード（`muroto_dashboard.html`）

ブラウザで直接開くだけで海流データを可視化できる単一HTMLファイル。外部のWebサーバーは不要で、`file://` プロトコルでのローカル表示に対応。

#### 主要機能

**① ダッシュボード（折りたたみ式）**
- 最新データ日付の表示
- 全5地点の最新流速・水温・塩分のカード表示
- データ件数・期間の統計情報
- 折りたたみボタンで表示/非表示を切り替え可能

**② 5地点データビューア**
- 地点タブ切り替え: 北 → 東 → 中心（デフォルト） → 西 → 南
- 期間選択: 7日 / 14日 / 30日（デフォルト）/ 90日 / 365日 / 全期間
- **流速・水温チャート**: 流速（kn）と水温（℃）の2軸グラフ
  - データポイントに方向矢印を表示（流向を直感的に把握）
  - 方位記号（N付き円形コンパス）をチャート右上に配置
- **塩分濃度チャート**: 塩分（psu）の時系列グラフ
- **データテーブル**: スクロール可能な日別データ一覧（ヘッダー固定）

**④ 欠損データチェッカー**
- 地点別の欠損日一覧表示
- 欠損なしの場合は「欠損なし」と表示

#### 技術仕様

| 項目 | 仕様 |
|---|---|
| ファイル形式 | 単一HTML（CSS/JS内蔵） |
| チャートライブラリ | Chart.js v4.4.0（CDN読み込み） |
| データ読み込み | `muroto_dashboard_data.js` をstatic `<script>`タグで読み込み |
| 動作環境 | モダンブラウザ（Chrome/Edge/Firefox） |
| 通信 | 不要（完全オフライン動作、CDN以外） |

#### 矢印表示の技術詳細

チャート上のデータポイントに流向を示す矢印を表示。以下の仕組みで実装:

- Canvas要素をキャッシュ（10°刻み、最大36種×サイズ別）して描画パフォーマンスを確保
- 正方形Canvas（Chart.jsの`pointStyle`仕様に合わせ歪み防止）
- 期間に応じて矢印サイズを自動調整:
  - 7日以内: 40px解像度 / 半径12
  - 14日以内: 36px / 半径11
  - 30日以内: 30px / 半径9
  - それ以上: 24px / 半径6

#### 方位記号

流速チャートの右上に配置されたSVG方位記号。Nの文字と円内の北向き三角矢印で、データポイントの矢印方向を直感的に理解できるようにしている。

### 2. データ変換スクリプト（`update_dashboard_data.py`）

CSVデータをJavaScript変数に変換し、HTMLダッシュボードが読み込めるようにするスクリプト。

| 項目 | 内容 |
|---|---|
| 入力 | `jcope_muroto/output/muroto_current_all.csv` |
| 出力 | `muroto_dashboard_data.js`（`window.MUROTO_CSV_TEXT`にCSV全文格納） |
| エスケープ処理 | バッククォート・バックスラッシュ・テンプレートリテラル |
| 実行方法 | `python update_dashboard_data.py` |

**file:// プロトコル対策:**
ブラウザのセキュリティにより `fetch()` や動的スクリプト挿入はローカルファイルで動作しないため、Pythonスクリプトで事前にJSファイルを生成し、HTMLの `<head>` 内でstatic `<script>` タグで読み込む方式を採用。

## 日常の運用手順

### 自動運用（設定済み）
1. 毎朝6:00にCoworkスケジューラーが `main.py --all` を実行
2. 新しい日付のデータが `muroto_current_all.csv` に追記される

### ダッシュボードの更新
1. CSVが更新された後、`update_dashboard_data.py` を実行
2. `muroto_dashboard_data.js` が再生成される
3. ブラウザで `muroto_dashboard.html` を開く（またはリロード）

```bash
# 手動で全工程を実行する場合
cd jcope_muroto
python main.py --all
cd ..
python update_dashboard_data.py
# → ブラウザで muroto_dashboard.html を開く
```

## v2.0開発で解決した主な課題

| 課題 | 原因 | 解決策 |
|---|---|---|
| `fetch()` がローカルで動かない | ブラウザの `file://` セキュリティ制限 | Python → JS変換 + static `<script>` 読み込み |
| 動的`<script>`挿入もブロック | Chromeの`file://`セキュリティ | `<head>`内のstatic `<script src>` で対応 |
| 矢印の三角形では方向が曖昧 | 正三角形は120°回転で同一形状 | カスタム矢印Canvas（矢じり＋シャフト形状） |
| 矢印の東西方向が異常に短い | 非正方形Canvasが`pointStyle`で歪む | 正方形Canvas + 内接描画に変更 |
| 塩分チャート・テーブルが消える | `renderCharts()`のエラーで後続処理が全停止 | 各レンダー関数を独立try-catchで分離 |
| Canvas要素が大量生成される | データ点ごとに新規Canvas作成 | 10°刻みキャッシュ方式（最大36枚/サイズ） |
| UTF-8 BOM でCSVパース失敗 | CSVの先頭に`\uFEFF`が付加 | `text.replace(/^\uFEFF/, '').trim()` で除去 |
