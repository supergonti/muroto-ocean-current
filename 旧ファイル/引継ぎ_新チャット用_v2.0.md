## プロジェクト概要

室戸岬沖の海流データ（流速・流向・水温・塩分）を日次で継続取得するPythonシステムと、それをブラウザで可視化するHTMLダッシュボードのセット。

_CMEMS_DATASET_AFC_TMP = "cmems_mod_glo_phy-thetao_anfc_0.083deg_P1D-m" # 水温(thetao)

## 今後の開発計画（v3.0に向けて）

### 最優先: GitHub移行とWeb公開

現在はローカル（`file://`）でのみ動作するシステムを、ネット経由でスマホなどからも閲覧できるようにする。

#### Step 1: GitHubリポジトリ作成

```
推奨リポジトリ名: muroto-ocean-current
推奨構成:
  muroto-ocean-current/
  ├── README.md
  ├── .gitignore              # output/cmems/, __pycache__, .copernicusmarine-credentials
  ├── jcope_muroto/
  │   ├── config.py
  │   ├── downloader.py
  │   ├── processor.py
  │   ├── main.py
  │   └── requirements.txt
  ├── dashboard/
  │   ├── index.html          # muroto_dashboard.html をリネーム
  │   └── data.js             # muroto_dashboard_data.js をリネーム
  └── scripts/
      └── update_dashboard_data.py
```

**注意事項:**
- `.copernicusmarine-credentials` は絶対にコミットしない（`.gitignore` に追加）
- `output/cmems/` のNetCDFキャッシュ（数百MB）もコミットしない
- `muroto_dashboard_data.js`（約608KB）をコミットするかは要検討（CSVデータそのものを含む）
- CSVファイル（607KB）は含めてもよいが、毎日更新されるためLFS検討

#### Step 2: GitHub Pages でダッシュボード公開

最もシンプルな方法:

1. GitHubリポジトリの Settings > Pages > Source を「GitHub Actions」または「Deploy from a branch」に設定
2. `dashboard/` フォルダ内の `index.html` + `data.js` を公開
3. `https://<username>.github.io/muroto-ocean-current/dashboard/` でアクセス可能に

**改修が必要な点:**
- 現在の `<script src="./muroto_dashboard_data.js">` のパスを `data.js` に変更
- Chart.jsのCDN読み込みは現在のままで動作する
- レスポンシブ対応の追加（`<meta name="viewport">` は設定済み、CSSの調整が必要）

#### Step 3: スマホ対応（レスポンシブ改修）

現在のダッシュボードはPC向けレイアウト。スマホ対応には以下の改修が必要:

- ダッシュボードカードを1列表示に変更（`@media (max-width: 600px)`）
- チャートの高さを画面幅に応じて調整
- テーブルの横スクロール対応
- タブボタンのサイズ拡大（タッチ操作対応）
- フォントサイズの調整

#### Step 4: データ自動更新の自動化（GitHub Actions）

ローカルのCoworkスケジューラーからGitHub Actionsに移行:

```yaml
# .github/workflows/daily-update.yml の概要
name: Daily Ocean Data Update
on:
  schedule:
    - cron: '0 21 * * *'  # UTC 21:00 = JST 06:00
  workflow_dispatch:       # 手動実行も可能

jobs:
  update:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-python@v5
        with:
          python-version: '3.10'
      - run: pip install -r jcope_muroto/requirements.txt
      - run: python jcope_muroto/main.py --all
        env:
          CMEMS_USERNAME: ${{ secrets.CMEMS_USERNAME }}
          CMEMS_PASSWORD: ${{ secrets.CMEMS_PASSWORD }}
      - run: python scripts/update_dashboard_data.py
      - name: Commit and push
        run: |
          git add -A
          git commit -m "Daily data update $(date +%Y-%m-%d)" || true
          git push
```

**必要な準備:**
- CMEMSのユーザー名・パスワードをGitHub Secretsに登録
- `downloader.py` を環境変数からの認証に対応させる改修
- GitHub Actionsの無料枠: 月2,000分（十分余裕あり）

## 過去に発生した主なエラーと解決法

### v1.0（データ収集系）

| エラー | 原因 | 解決法 |
|---|---|---|
| `No module named h5py` | h5pyが未インストール | `pip install h5py --break-system-packages` |
| 水温・塩分がnanになる | 2026年用データセット選択ミス | 3データセットをmergeする方式に変更 |
| 複数のCSVファイルが作られる | 出力ファイル名が日付依存 | `_all.csv` に固定化 |

### v2.0（ダッシュボード系）

| エラー | 原因 | 解決法 |
|---|---|---|
| `fetch()` がエラー | `file://` セキュリティ | Python → JS変換 + static script |
| 矢印の方向が不明瞭 | 正三角形は120°対称 | カスタム矢印Canvas |
| 東西方向の矢印が短い | 非正方形Canvas | 正方形Canvasに統一 |
| チャートエラーで全画面真っ白 | 一つの関数内で全処理 | 独立try-catch分離 |
| CSVパースでヘッダー不一致 | UTF-8 BOM | `replace(/^\uFEFF/, '')` |
