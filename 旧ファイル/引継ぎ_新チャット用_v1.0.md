## 重要な技術仕様

### データソース切り替えロジック（downloader.py）

```python
# 2025年以前: GLORYS12再解析データ（1データセット、全変数入り）
GLORYS12 = "cmems_mod_glo_phy_my_0.083deg_P1D-m"

# 2026年以降: 3つの予報データセットを個別にDLしてxr.mergeで統合
_CMEMS_DATASET_AFC_CUR = "cmems_mod_glo_phy-cur_anfc_0.083deg_P1D-m"    # 流速(uo,vo)
_CMEMS_DATASET_AFC_TMP = "cmems_mod_glo_phy-thetao_anfc_0.083deg_P1D-m" # 水温(thetao)
_CMEMS_DATASET_AFC_SAL = "cmems_mod_glo_phy-so_anfc_0.083deg_P1D-m"     # 塩分(so)
```

切り替え判定: `_select_cmems_dataset(target_date)` で日付によって自動選択

### メインCSVの固定ファイル名（main.py）
```python
# 以前は日付範囲ベースのファイル名だったが、v1.0で固定化
output_path = output_dir / f"{OUTPUT_PREFIX}_all.csv"
# → output/muroto_current_all.csv に常に書き出す
```

### 測定地点（config.py）
```python
MEASUREMENT_POINTS = {
    "中心": {"lat": 33.2667, "lon": 134.2833},
    "北":   {"lat": 33.3567, "lon": 134.2833},
    "南":   {"lat": 33.1767, "lon": 134.2833},
    "東":   {"lat": 33.2667, "lon": 134.3933},
    "西":   {"lat": 33.2667, "lon": 134.1733},
}
```

### CSV列定義
```
date, point, lat, lon, u_ms, v_ms, speed_ms, speed_kn, direction, temp_c, salinity
```
- `speed_kn = speed_ms × 1.944`
- `direction`: 0=北、90=東、180=南、270=西（時計回り）

## 過去に発生した主なエラーと解決法（参考）

| エラー | 原因 | 解決法 |
|---|---|---|
| `No module named h5py` | h5pyが未インストール | `pip install h5py --break-system-packages` |
| 水温・塩分がnanになる | 2026年用データセット選択ミス | `_CMEMS_DATASET_AFC_CUR` は流速のみ。3データセットをmerge |
| 複数のCSVファイルが作られる | main.pyの出力ファイル名が日付依存だった | `OUTPUT_PREFIX_all.csv` に固定化済み |
| キャッシュが残って再取得できない | cmems/フォルダのncファイルが邪魔 | `--no-skip` オプションで上書き取得 |
| `force_download` deprecated warning | copernicusmarineライブラリの仕様変更 | パラメータを削除済み |
