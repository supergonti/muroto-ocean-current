# 室戸沖 海流データ収集システム

**バージョン: 1.0**
**作成日: 2026年4月12日**
**ステータス: 稼働中（自動収集スケジュール設定済み）**

---

## 概要

室戸岬沖（室戸沖）の海流データを2022年1月1日以降の日次データとして継続的に収集・蓄積するPythonシステム。
CMEMS（Copernicus Marine Service）のAPIを主データソースとして使用し、取得したNetCDF形式データをCSV形式に変換・蓄積する。

---

## システムの目的と方針

### 目的
- 室戸沖5地点における海流（流速・流向）・水温・塩分の日次データを継続取得
- 漁業ログ開発のための海況データベース構築
- 2022年1月1日から現在まで、毎日自動更新

### 基本方針
1. **無料・継続利用可能なデータソース**を優先（CMEMS無料登録）
2. **単一のCSVファイル**に全データを蓄積し、常に上書き更新
3. **毎朝6時の自動実行**で前日データを自動追加
4. 将来的にはJAMSTECのJCOPE2Mへ移行も検討（申請制）

| データソース | 種別 | 対象期間 | データセットID |

> **注記**: 2026年以降は流速・水温・塩分が別データセットに分かれているため、3データセットをダウンロード後にマージして使用。

## 出力データ仕様（CSVカラム定義）

| カラム名 | 単位 | 説明 |
|---|---|---|
| date | YYYY-MM-DD | 日付 |
| point | 文字列 | 地点名（中心/北/南/東/西） |
| lat | 度 | 緯度 |
| lon | 度 | 経度 |
| u_ms | m/s | 東西流速（東向き正） |
| v_ms | m/s | 南北流速（北向き正） |
| speed_ms | m/s | 流速（合成） |
| speed_kn | ノット | 流速（合成、ノット換算） |
| direction | 度 | 流向（0=北、時計回り） |
| temp_c | °C | 表層水温 |
| salinity | PSU | 塩分 |

**現在のデータ規模（v1.0時点）**
- 期間: 2022年1月1日〜2026年4月12日（約4年4ヶ月）
- 行数: 7,815行（5地点 × 1,563日）
- ファイルサイズ: 約607KB

---

## 計算方式

### 流速合成
```
speed_ms = √(u² + v²)
speed_kn = speed_ms × 1.944
```

### 流向計算
```
direction = atan2(u, v) × (180 / π)
（0°=北、90°=東、180°=南、270°=西、時計回り）
```

## ファイル説明

### `config.py`
- 測定地点5地点の座標定義
- データソース設定（CMEMS/JCOPE切り替え）
- 出力設定（ファイル名・カラム定義）

### `downloader.py`
- CMEMS APIへの接続・データ取得
- 2025年以前: GLORYS12再解析データ（1データセット）
- 2026年以降: ANFC予測データ（3データセットをxr.mergeで統合）
- NetCDFキャッシュ管理（再取得スキップ）

### `processor.py`
- NetCDFデータから5地点の値を抽出
- 流速・流向の計算
- CSVへの追記・重複除去

### `main.py`
- コマンドラインインターフェース
- 主要オプション:
  - `--all`: 全期間（2022/1/1〜今日）を取得
  - `--start YYYY-MM-DD --end YYYY-MM-DD`: 期間指定
  - `--date YYYY-MM-DD`: 1日分のみ取得
  - `--no-skip`: キャッシュを無視して再取得
  - `--check`: 欠損日チェック
  - `--summary`: 月次サマリー生成

## 自動実行スケジュール

**毎朝6:00（ローカル時間）に自動実行**
- スケジューラー: Cowork スケジュールタスク
- 実行内容: 前日までのデータを `muroto_current_all.csv` に追記
- 出力ファイル: 常に同じファイル名（上書き更新でなく追記）

---

## データの蓄積方式

```
初回実行（2022/1/1〜最新）
       ↓
muroto_current_all.csv に全データ書き込み
       ↓
毎朝6時の自動実行
       ↓
新しい日付のデータを追記（既存データは保持）
       ↓
重複除去処理（同じ日付・地点が重複しないよう自動排除）
```

## 既知の制限・注意事項

1. **2026年以降のデータ**: 3つの別データセットをマージするため、取得に時間がかかる
2. **CMEMSのAPI制限**: 大量リクエストの場合はレート制限がかかる場合がある
3. **キャッシュ**: `output/cmems/` にNetCDFファイルをキャッシュ。再取得時は `--no-skip` オプション使用
4. **JCOPE2Mへの移行**: JAMSTEC申請後は `config.py` の設定変更で切り替え可能
